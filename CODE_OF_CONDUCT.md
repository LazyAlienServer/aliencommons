# Code of Conduct

This Code of Conduct applies to participation in the AlienCommons open source project, including GitHub issues, pull requests, discussions, reviews, commit messages, documentation contributions, and related project communication channels.

AlienCommons itself is a Minecraft-focused community product. The same core values guide the development community: keep the project focused, constructive, safe, and free from disruptive commercial noise.

## Expected Behavior

Contributors are expected to:

- Keep discussions relevant to the project, Minecraft-community product needs, implementation details, documentation, operations, or other directly useful work.
- Be respectful when disagreeing about design, code, product direction, or moderation policy.
- Give and receive feedback in good faith.
- Respect maintainer decisions about scope, priorities, and project safety.
- Avoid derailing technical discussions with unrelated debates or promotional content.

## Unacceptable Behavior

The following behavior is not acceptable in the project community:

- Political, ideological, or highly sensitive social dispute content.
- Pornographic, sexually explicit, or sexualized material.
- Violent, graphic, threatening, or dangerous material.
- Religious controversy, proselytizing, or other disruptive religious debate.
- Hate, discrimination, dehumanization, or attacks against protected or identity-based groups.
- Illegal activity, scams, doxxing, credential theft, or instructions that enable harm.
- Personal attacks, harassment, bullying, intimidation, stalking, or targeted abuse.
- Spam, irrelevant promotion, advertising, link farming, bot-like posting, or repeated off-topic noise.
- Bad-faith participation, including intentionally wasting maintainer time or repeatedly reopening settled disputes.

## Reporting

If you see behavior that violates this Code of Conduct, contact the maintainers privately when possible. Do not open public issues for sensitive personal reports, security-sensitive information, or private abuse reports.

For security vulnerabilities, follow [SECURITY.md](SECURITY.md) instead of using public GitHub issues.

## Enforcement

Maintainers may edit or delete comments, close issues or pull requests, restrict participation, block users, or take other action needed to protect the project. Enforcement decisions should preserve a focused and healthy development environment while allowing ordinary good-faith disagreement about technical and product decisions.
